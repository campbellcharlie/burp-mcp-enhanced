=== Vulnerability Detection Patterns ===

Response patterns indicating vulnerabilities. Use these to identify successful exploitation.

--- SQL INJECTION ---

  MySQL errors:
    You have an error in your SQL syntax
    Warning: mysql_  /  Warning: mysqli_
    Column count doesn't match  /  Unknown column
    Table '.+' doesn't exist

  PostgreSQL errors:
    ERROR: syntax error at or near
    ERROR: unterminated quoted string
    pg_query()  /  pg_exec()  /  PSQLException

  Oracle errors:
    ORA-00933: SQL command not properly ended
    ORA-00936: missing expression
    ORA-01756: quoted string not properly terminated

  MSSQL errors:
    Unclosed quotation mark  /  Incorrect syntax near
    Conversion failed  /  Microsoft OLE DB Provider
    ODBC SQL Server Driver  /  SqlException

  SQLite errors:
    SQLITE_ERROR  /  SQLite3::
    near "": syntax error  /  unrecognized token

  Successful extraction:
    Version strings: 5.7.42, PostgreSQL 13.4, Oracle 19c, SQL Server 2019
    Schema names: information_schema, sysobjects, all_tables
    Data: root:x:0:0:, admin:password123

--- XSS (CROSS-SITE SCRIPTING) ---

  Success: Payload appears unencoded in response
    <script>alert(1)</script>
    <img src=x onerror=alert(1)>
    <svg/onload=alert(1)>

  Attribute context:
    " onmouseover="alert(1)
    ' onfocus='alert(1)

  Partial success (filter bypass needed):
    Events work when <script> blocked
    Payload reflected but encoded
    Payload inside JavaScript string

  IMPORTANT: Only proof = alert fires in actual browser.

--- SSTI (SERVER-SIDE TEMPLATE INJECTION) ---

  Mathematical evaluation:
    {{7*7}} -> 49        (Jinja2, Twig, Handlebars)
    ${7*7}  -> 49        (Freemarker, Velocity)
    <%= 7*7 %> -> 49     (ERB)

  String manipulation:
    {{'7'*7}} -> 7777777     (Jinja2/Twig)

  Engine identification via errors:
    jinja2.exceptions.UndefinedError           -> Jinja2
    Twig_Error_Syntax / Twig_Error_Runtime     -> Twig
    freemarker.core.InvalidReferenceException  -> Freemarker
    org.apache.velocity.exception              -> Velocity
    SyntaxError: (erb)                         -> ERB

  Successful code execution:
    uid=33(www-data) gid=33(www-data)
    SECRET_KEY=, DATABASE_URL=
    <class 'subprocess.Popen'>

--- COMMAND INJECTION ---

  Success indicators:
    uid=33(www-data) gid=33(www-data) groups=33(www-data)
    root:x:0:0:root:/root:/bin/bash
    Linux hostname 5.4.0-42-generic

  Blind detection:
    Response time increases ~5s after: ; sleep 5, | sleep 5
    OOB callback: DNS/HTTP to attacker domain

  Error indicators:
    sh: 1: invalid_command: not found
    Permission denied / Operation not permitted

--- XXE (XML EXTERNAL ENTITY) ---

  Success:
    File contents: root:x:0:0:root:/root:/bin/bash, [boot loader]
    Internal service response embedded in output
    Error-based: java.io.FileNotFoundException

  Blind XXE:
    OOB DNS/HTTP callback received
    File contents appear in error message

  Parser identification:
    org.xml.sax.SAXParseException       -> Java SAX
    javax.xml.stream.XMLStreamException -> Java StAX
    lxml.etree.XMLSyntaxError           -> Python lxml
    SimpleXMLElement / DOMDocument      -> PHP

--- PATH TRAVERSAL ---

  Linux success: root:x:0:0:root:/root:/bin/bash
  Windows success: [fonts] / [extensions] / [mci extensions]
  Source disclosure: <?php, import, require, def __init__

--- SSRF (SERVER-SIDE REQUEST FORGERY) ---

  Internal service access:
    Response content from localhost (Apache, nginx)
    Admin Dashboard / Internal API Response
    Status code changes: external=timeout, internal=200

  Cloud metadata:
    AWS:   {"AccessKeyId":"ASIA...","SecretAccessKey":"...","Token":"..."}
    GCP:   {"access_token":"ya29...","token_type":"Bearer"}
    Azure: {"access_token":"eyJ0...","resource":"https://management.azure.com/"}

  Cloud metadata endpoints:
    AWS:   http://169.254.169.254/latest/meta-data/iam/security-credentials/
    GCP:   http://metadata.google.internal/computeMetadata/v1/ (Header: Metadata-Flavor: Google)
    Azure: http://169.254.169.254/metadata/instance?api-version=2021-02-01 (Header: Metadata: true)

  OOB indicators:
    DNS query from target server IP
    HTTP request with server User-Agent

--- JWT VULNERABILITIES ---

  None algorithm: Token accepted without signature, user context changed
  Algorithm confusion: Token signed with public key accepted
  Weak secret: Brute force successful, forged token accepted

--- RACE CONDITIONS ---

  Success: Multiple transactions succeed, negative balance, duplicate coupon

--- RESPONSE COMPARISON ---

  Baseline vs Injected:
    Status:  200 -> 500 (error-based)
    Length:  1500 -> 2500 (data extracted)
    Time:   100ms -> 5100ms (time-based blind)
    Content: Product -> Error message

  Boolean conditions:
    True:  200, length 1500, "Product found"
    False: 200, length 1200, "No results"
    -> If different = boolean injection confirmed

--- QUICK REFERENCE ---

  Vulnerability         Key Pattern
  -------------         -----------
  SQL Injection         SQL error messages, unexpected data
  XSS                   Payload reflected unencoded (verify in browser)
  SSTI                  {{7*7}} = 49 in response
  Command Injection     Command output (uid=, whoami)
  XXE                   File contents, OOB callback
  Path Traversal        root:x:0:0 or [fonts]
  SSRF                  Internal responses, cloud metadata
  JWT                   Token accepted after tampering
  Race Condition        Duplicate transactions
